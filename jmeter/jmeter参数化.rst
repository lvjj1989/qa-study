Jmeter参数化
==========================================

CSV Data Set Config
--------------------------------------

CSV Data Set Config 可以指定文本文件，并一行一行地提取文本内容，根据分隔符拆解这一行内容与变量名对应，然后这些变量就可以提供给取样器引用。

参数说明如下：

**名称**：随意设置，最好有业务意义

**注释**：随意设置，可以为空

**Filename**：参数文件地址，可以是相对路径，也可以是绝对路径。相对路径的根节点是 JMeter 的启动目录（%JMETER_HOME%\\bin）

**File encoding**：读取参数文件的编码格式，建议使用UTF-8

**Variable Names**：定义参数名称，用逗号隔开，将会与文件中的参数列一一对应。如果这里的参数数量超过了文件中的列数，多余的参数将会取不到值。例如：loginname, passWord，在 JMeter 中读取该参数的方式为 `${loginname}`、`${passWord}`。

**Delimiter**：用于分割文件的分隔符，默认是英文逗号。如果参数文件使用的是制表符分割，则输入：`\t`

**Allow quoted data?**：是否允许被引号包裹的数据中包含分隔符，如果选择“是”，则允许

**Recycle on EOF?**：是，参数文件循环遍历；否，参数文件遍历完成后不再循环。

**Stop thread on EOF?**：与 Recycle on EOF 的 False 组合使用；是：到达文件末尾时停止线程；否：到达文件末尾时线程继续执行但不再从文件读取数据。

**Sharing mode**：参数文件共享模式，有以下几种：

* All threads：参数文件对所有线程共享，这就包括同一测试计划中的不同线程组；
* Current thread group：只在当前线程组内共享；
* Current thread：只在当前线程内使用。
